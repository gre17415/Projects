# Философы

За круглым столом сидят `n` философов. Перед каждым из них стоит тарелка. Между каждой парой соседних философов лежит вилка.

Каждый философ за столом действует по следующему алгоритму:

1) Берет две лежащие рядом вилки
2) Ест пасту
3) Кладет вилки обратно на стол
4) Размышляет над чем-то
5) Затем повторяет все сначала

Отсюда следует, что два сидящих рядом филосфа не могут есть одновременно.
Нужно научить философов брать и класть вилки, чтобы они не могли заблокировать друг друга.

Реализуйте методы `AcquireForks` и `ReleaseForks` таким образом, чтобы дэдлок гарантированно не мог возникнуть.
А также напишите реализацию для Fork.

Можно модифицировать файлы fork.h, philosopher.cpp.

Использовать функцию [`std::lock`](https://en.cppreference.com/w/cpp/thread/lock) запрещается!

## Deadlock prevention / avoidance

Если в своем решении вы используете метод мьютекса `try_lock`, то скорее всего вы реализовали _deadlock avoidance_, а не _deadlock prevention_.

В решении с _deadlock prevention_ нужно использовать только методы `lock` и `unlock`.
